# 接口组织建议

## 一、核心变换接口

### 1.1 `Transform` — 仿射变换类

路径: `core/transform.ts`

```ts
class Transform {
  constructor(a = 1, b = 0, tx = 0, c = 0, d = 1, ty = 0)
  // 矩阵: | a  c  tx |
  //       | b  d  ty |
  //       | 0  0  1  |

  applyTo(x, y)        → { x, y }           // 变换单个点
  applyToArray(p)      → [number, number]   // 变换坐标数组
  multiply(t)          → Transform          // 矩阵乘法复合
  toArray()            → [[a,c,tx],[b,d,ty]]
  static identity()    → Transform
  static translation(pt) → Transform
  static scaling(sx, sy = sx) → Transform
  static rotation(rad) → Transform
  static rotationAbout(center, rad) → Transform
  static mirrorX()     → Transform
  static mirrorY()     → Transform
  static mirrorLine(rad) → Transform
  static mirrorAbout(pt, rad) → Transform
  static mirrorAboutLine(line) → Transform
  static degToRad(deg) → number
  static radToDeg(rad) → number
  static fromArray(m?) → Transform
}
```

### 1.2 `Entity` — 几何与变换接口

路径: `core/entity.ts`

```ts
class Entity {
  // ── 几何获取 ──
  getRepresent(resolver)          → Point2d          // 实体的代表点
  getResult(resolver)             → Point2d          // 同 getRepresent
  getRefPoint(resolver, pt)       → Point2d          // 代表点 + pt 偏移
  getBox(resolver)                → Box              // 包围盒
  getCurve(resolver)              → Curve | null     // 曲线对象
  length(resolver)                → number           // 曲线长度
  getSnapPoints(mode, resolver)   → SnapPoint[] | null
  resolveRef(ref, resolver)       → Point2d | null   // 解引用点

  // ── 变换操作 ──
  applyTransform(resolver, t: Transform) → void      // 应用仿射变换
  update(resolver)                → void              // 清除缓存 (_curve = null)
  clone()                         → Entity            // 深拷贝（使用注入的工厂）
  explode(resolver)               → Entity[] | null   // 分解为子实体
  matchProperties(source)         → void              // 复制绘制属性

  // ── 夹点交互 ──
  getGripPoints(resolver)         → GripPoint[] | null
  onGripDrag(propPath, newX, newY, resolver) → boolean

  // ── 属性编辑 ──
  getProperties()                 → PropertyItem[]
  setProperty(key, value, resolver) → boolean

  // ── 序列化 ──
  toJSON()                        → Record<string, unknown>
  static get cliCommands()        → CmdDesc[]
}
```

### 1.3 批量变换

路径: `core/barrel.ts`

```ts
// 对一组实体逐一应用仿射变换（平移/旋转/缩放/镜像）
batchTransform(resolver, entities, t: Transform) → void

// 对一组实体按镜像线做镜像
batchMirror(resolver, entities, p1, p2) → void
```

### 1.4 依赖追踪

路径: `core/resolver.ts`

```ts
class Resolver implements IResolver {
  doc: GSGIDocument
  entityCache: Map<string, Entity>   // id → Entity
  _dependents: Map<string, Set<string>>   // 引用链

  _buildCache()        → void         // 重建 entityCache
  _buildDeps()         → void         // 重建依赖图
  _trackEntityDeps(id, entity) → void // 追踪单实体的引用
  updateItems(ids)     → void         // 更新一组实体（重建缓存+触发重绘）
  get(id)              → Entity | null
}
```

`IResolver` 接口定义见 `core/entity.ts`，供 `core/entities/*` 子类使用。

## 二、变换操作 & 数据流

### 2.1 操作 → UI → Core 链路

```
用户操作（拖拽/命令栏/快捷键）
    ↓
Viewer 事件处理器（_setupEvents / _processCommand）
    ↓
DrawToolController / TransformToolController
    ├─ move:    drawMoveClick → Transform.translation → batchTransform → updateItems
    ├─ copy:    deepCloneEntity + drawMoveClick → batchTransform
    ├─ rotate:  drawRotateClick → Transform.rotationAbout → batchTransform → updateItems
    ├─ mirror:  drawMirrorClick → batchMirror → updateItems
    ├─ erase:   直接 doc.entities.filter → finishDraw
    └─ grip:    _doGripDrag → onGripDrag → _endGripDrag → updateItems
```

所有变换工具（移动/复制/旋转/对称）统一采用两步式：

- **预选**：选中实体 → 点击工具 → 进入绘图状态（step 1）
- **无预选**：点击工具 → 选择模式（step 0） → 选中后回车 → 进入绘图状态（step 1）

| 工具 | step 1 | step 2 |
|------|--------|--------|
| 移动/复制 | 点击基点 | 点击终点 |
| 旋转 | 点击中心点 | 输入角度（键盘或点击） |
| 对称 | 点击轴第一点 | 点击轴第二点 |

选择行为：单击替换选中；Shift+单击加选/减选；左拖框选（包含）、右拖框选（相交）。

### 2.2 TransformToolController

路径: `viewer/controllers/TransformToolController.ts`

控制移动/复制/旋转/镜像的交互：

```ts
class TransformToolController {
  // 各操作入口（被 DrawToolController.handleDrawClick 委托调用）
  drawMoveClick(pt, rawPt)       → void   // 移动：step0=点击选择, step1=基点, step2=终点
  drawCopyClick(pt, rawPt)       → void   // 复制：同移动, 但克隆后再移动
  drawRotateClick(pt, rawPt)     → void   // 旋转：step0=选择, step1=中心点, step2=角度(键盘/点击)
  drawMirrorClick(pt, rawPt)     → void   // 镜像：step0=选择, step1=第一点, step2=第二点

  // 预览
  updateMovePreview(pt)          → void
  updateCopyPreview(pt)          → void
  revertMovePreview()            → void
  removeCopyPreview()            → void

  // 提交
  finishTransform(entity, before, prompt) → void

  // 键盘角度输入
  applyRotateByAngle(angleDeg)   → void

  // 工具
  cloneEntities(ids)             → { copies, oldToNew }  // 批量克隆（含依赖点）
}
```

### 2.3 夹点拖拽

在 Viewer 中直接管理夹点拖拽生命周期：

```
_hitTestGrip(x, y)  → boolean    // 检测点击到夹点
_startGripDrag(x, y) → boolean   // 开始拖拽
_doGripDrag(x, y)    → void      // 拖拽中：调用 entity.onGripDrag()
_endGripDrag()       → void      // 结束：提交撤销快照
_removeGrips()       → void      // 隐藏夹点
```

每个实体子类实现 `onGripDrag()` 定义自己的夹点行为（移动点引用、调整半径等）。

### 2.4 撤销/重做

使用快照模式，路径: `viewer/util/clipboard.ts` + `Viewer`

```ts
// 生成快照
const before = viewer._saveSnapshot()  // = cloneDocumentData(doc)

// 操作后，通过工厂函数推入撤销栈
viewer._undoManager.push(createUndoCommand(entityId, before, after, viewer))

// 工具函数（viewer/util/clipboard.ts）
cloneDocumentData(doc)                    → unknown       // 深拷贝文档
applyDocumentData(viewer, data)           → void           // 恢复文档
createUndoCommand(entityId, before, after, viewer) → {...} // 撤销命令工厂
```

## 四、渲染管线

### 4.1 架构

```
Renderer.render()
  ├─ _collectReferencedPoints()     // 收集被引用的构造点
  ├─ buildLayerGroups(...)           // 构建图层分组（render/layer-manager.ts）
  ├─ for each entity: _renderEntity(entity)
  │    ├─ _buildFrozenSet()         // 计算冻结层集合
  │    ├─ getTargetLayer(...)       // 确定目标图层（render/layer-manager.ts）
  │    └─ _renderEntityToParent(entity, layer)
  │         └─ _dispatchers[entity.type]  → dispatcher(entity, layer)   // RenderVisitor
  └─ _registerEntityItems(entity, items)   // 注册到 itemMap / entityItems
```

### 4.2 RenderVisitor

路径: `render/render-visitor.ts`

```ts
registerDispatchers(renderer) → void
  // 渲染器构造函数末尾调用，填充 renderer._dispatchers
  // 每种实体类型注册一个 (entity, parent) → any[] | null 处理器
```

### 4.3 Renderer

路径: `render/renderer.ts`

```ts
class Renderer {
  doc: GSGIDocument
  resolver: Resolver
  _dispatchers: Record<string, (entity, parent) => any[] | null>  // RenderVisitor 注册

  // 核心方法
  clear()                                   → void               // 清除所有图层
  render()                                  → void               // 全量重绘
  renderBlock(blockDef, parent, resolver, doc) → paper.Group     // 块渲染

  // 分发
  _renderEntityToParent(entity, parent)     → paper.Item[] | null
  _registerEntityItems(entity, items)       → void               // 注册映射

  // 样式工具
  _color(entity)                            → string             // 解析实体颜色

  // 屏幕固定元素
  _createScreenFixedItem(kind, x, y, sizePx, style) → paper.Item

  // 后续更新
  updateStrokeWidths()                      → void               // 缩放后更新线宽
  updateScreenFixedItems()                  → void               // 缩放后更新

  // 引用点收集
  _collectReferencedPoints()                → Set<string>
}
```

### 4.4 颜色解析

路径: `core/color-resolver.ts`

```ts
class ColorResolver {

  static resolveColorValue(c: number | string | undefined): string  // 颜色值 → hex
```

## 五、实体数据流

### 5.1 创建

```
用户点击 → handleDrawClick
  → DRAW_HANDLER_MAP[tool](viewer, pt)                  // 策略分发
  → viewer._ensurePoint(pt, pt, desc) → pointId         // 创建/复用构造点（含 construction 标记）
  → viewer._createEntity({ type, id, ...refs })          // 自动填充 scale
  → doc.entities.push(entity)
  → _finishDraw(entity, before)                         // 快照 + 撤销栈
  → Renderer.render()                                   // 全量重新渲染
```

### 5.2 编辑

```
属性面板 → entity.setProperty(key, value, resolver)
  → resolver.updateItems([entity.id])                   // 清除缓存 + 触发重绘
```

夹点拖拽:
```
entity.onGripDrag(propPath, newX, newY, resolver) → boolean
  → resolver.updateItems([entity.id])                   // 同上
```

### 5.3 批量变换

```
TransformToolController:
  Transform.translation(dx, dy) → t
  batchTransform(resolver, entities, t)
    // 对每个 entity:
    //   entity.applyTransform(resolver, t)   → 移动引用点
    //   entity.update(resolver)              → 清除 _curve 缓存
    //   resolver.updateItems(ids)            → 重建缓存 + 触发重绘
```

### 5.4 依赖复制

```
TransformToolController.deepCloneEntity(entity):
  // 1. 递归查找引用的 point 实体
  // 2. 深拷贝所有依赖点
  // 3. 更新新实体的引用指向克隆后的点
  // 4. 返回新的独立实体
```

## 六、关键子模块

| 模块 | 路径 | 职责 |
|------|------|------|
| ColorResolver | `core/color-resolver.ts` | ACI ↔ hex、ByLayer 解析 |
| Resolver | `core/resolver.ts` | 实体缓存、依赖追踪、级联更新 |
| paper-utils | `render/paper-utils.ts` | makePath、applyStyle、linetypeDashArray |
| layer-manager | `render/layer-manager.ts` | buildLayerGroups、setWorldLayer、getTargetLayer |
| screen-fixed | `render/screen-fixed.ts` | 屏幕固定元素创建/更新 |
| draw-handlers | `viewer/tools/draw-handlers.ts` | 绘制 handler 策略模式 |

| `DrawData` | `viewer/controllers/DrawToolController.ts` | 绘制过程临时数据 |
