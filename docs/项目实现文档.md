# GSGI-tool 项目实现文档 / Project Implementation Document

## 1. 项目概述 / Project Overview

### 1.1 目标 / Objectives

基于 [GSGI](https://github.com/adgram/GSGI)（General Simple Geometry Information）数据模型，提供轻量级的 Web 查看器/编辑器和格式转换工具链。
A lightweight web viewer/editor and format conversion toolchain based on the [GSGI](https://github.com/adgram/GSGI) data model.

1. **Web 查看器/编辑器 / Web Viewer/Editor**：基于 Paper.js 的轻量级 GSGI 文件浏览和基础编辑 / Paper.js-based GSGI file viewer with basic editing capabilities
2. **格式转换器 / Format Converter**：与 DXF 等工业格式双向转换（计划中）/ Bidirectional conversion with DXF and other industrial formats (planned)

### 1.2 整体架构 / Overall Architecture

```
                    ┌─────────────────────────────────┐
                    │   Web 编辑器（Paper.js）         │
                    │   Web Editor (Paper.js)          │
                    │   — 查看/编辑 GSGI 文件          │
                    │   — View/Edit GSGI files         │
                    │   — 引用链实时求值               │
                    │   — Real-time reference eval     │
                    │   — 交互（选择/移动/属性）       │
                    │   — Interaction (select/move/prop)│
                    └──────────┬──────────────────────┘
                               │ .gsgi (JSON)
                    ┌──────────▼──────────────────────┐
                    │   格式转换层（Python，计划中）    │
                    │   Format Converter (Python, planned)
                    │   — DXF ↔ GSGI                  │
                    └──────────┬──────────────────────┘
                               │ .dxf
                    ┌──────────▼──────────────────────┐
                    │   外部 CAD 工具链                │
                    │   External CAD Toolchain         │
                    └─────────────────────────────────┘
```

## 3. 格式转换层详细设计 / Format Converter Detailed Design

### 3.1 技术选型 / Technology Selection

| 项 / Item | 选择 / Choice | 理由 / Rationale |
|----|------|------|
| DXF 读写 / DXF I/O | `ezdxf`（Python） | 最成熟的 Python DXF 库，支持 R2010+，组码完整 / Most mature Python DXF library, full group code support |
| DWG 中转 / DWG Bridge | ODA File Converter（免费 / Free） | 命令行批量转 DXF / Batch CLI conversion to DXF |
| Rhino .3dm | `rhino3dm`（Python） | 无需 Rhino 许可，OpenNURBS 内核 / No Rhino license required, OpenNURBS kernel |
| CLI 框架 / CLI Framework | `click` | 命令行参数解析 / Command-line argument parsing |

### 3.2 转换映射总纲 / Conversion Mapping Overview

#### DXF → GSGI

```
DXF                           GSGI
────                           ────
HEADER section                → properties
LAYERS table                  → layers[]
BLOCK_RECORD / BLOCKS         → blocks[]
ENTITIES section:
  POINT                       → point
  LINE                        → line
  LWPOLYLINE                  → polyline / polyarc（含 bulge 检测 / with bulge detection）
  CIRCLE                      → circle
  ARC                         → arc
  TEXT                        → text
  MTEXT                       → mtext
  INSERT（块参考 / Block Ref）  → block_ref
  SPLINE                      → spline_fit / spline_cv
  HATCH                       → hatch
  DIMENSION                   → dimension
  MLINE                       → mline
  XREF                        → xref
  ELLIPSE                     → polycurve（降级 / Degraded）
  LEADER / MULTILEADER        → polyline + text
  TABLE                       → table
  VIEWPORT（布局 / Layout）    → space: "paper"
  ⋮

扩展数据（XDATA）              → descriptions[] / property
```

#### GSGI → DXF

反向映射，要注意：
Reverse mapping, noting:

- `param_pt` → DXF 无等效实体，输出为 `POINT` + XDATA 标记 / No DXF equivalent, output as `POINT` + XDATA marker
- `position` → DXF 无等效实体，输出为 XDATA（应用层解释）/ No DXF equivalent, output as XDATA
- `polycurve` → `LWPOLYLINE`（直线/圆弧段保留），`subsegment_ref`/`curve_ref` 展开为具体路径 / Line/arc segments preserved, refs expanded
- `region_anno` → `LWPOLYLINE` + XDATA 标记 / XDATA marker
- `coord_sys` → `UCS` 定义 + `POINT`（可选显示 / Optional display）
- `measure` → `POINT` 对 + XDATA（测量注释 / Measurement annotation）
- `custom_entity` → `LINE` 或 `LWPOLYLINE` + XDATA（按 `custom_type` 降级 / Degraded by type）

### 3.3 XDATA 编码约定 / XDATA Encoding Convention

GSGI 特有的语义信息通过 DXF 扩展数据（XDATA）存储：
GSGI-specific semantic information is stored via DXF extended data (XDATA):

| 应用名 / App Name | 说明 / Description |
|--------|------|
| `"GSGI_META"` | 通用语义元数据 / General semantic metadata |
| `"GSGI_DESC"` | 描述文字 / Description text |
| `"GSGI_REF"` | 引用关系（`ref_pt`、`curve_ref`）/ Reference relationships |
| `"GSGI_SUBSEG"` | subsegment 参数 / Subsegment parameters |
| `"GSGI_PARAMPT"` | param_pt 参数 / param_pt parameters |
| `"GSGI_POSITION"` | position 关系定义 / Position relationship definition |
| `"GSGI_REGION"` | region_anno 区域信息 / Region annotation info |

**XDATA 格式示例 / XDATA Format Example**：

```
(1001, "GSGI_REF")          ; 应用名 / App name
(1002, "EX")                ; 字符串：引用类型（ref_pt / curve_ref）/ Ref type string
(1000, "P1")                ; 字符串：目标实体 id / Target entity id
```

## 4. 关键算法备忘 / Key Algorithm Notes

### 4.1 Bulge → 圆弧 / Bulge → Arc

```
给定 bulge b，弦长 chord：
Given bulge b, chord length chord:
  θ = 4 · atan(|b|)          // 圆心角（弧度）/ Central angle (radians)
  r = chord / (2 · sin(θ/2)) // 半径 / Radius
  方向：b > 0 逆时针，b < 0 顺时针
  Direction: b > 0 CCW, b < 0 CW
  圆心 = 弦中垂线上距离 √(r² - (chord/2)²) 的点
  Center = point on perpendicular bisector at distance √(r² - (chord/2)²)
```

### 4.2 param_pt.t 在 polyline 上的求值 / param_pt.t Evaluation on Polyline

```
开放 polyline（N 个顶点，N-1 段）：
Open polyline (N vertices, N-1 segments):
  t ∈ [0, N-1]
  段索引 i = floor(t)，i ∈ [0, N-2]
  Segment index i = floor(t), i ∈ [0, N-2]
  段内位置 s = t - i，s ∈ [0, 1)
  Position within segment s = t - i, s ∈ [0, 1)
  坐标 = lerp(V[i], V[i+1], s)
  Coordinate = lerp(V[i], V[i+1], s)

闭合 polyline（N 个顶点，N 段，末段 = V[N-1]→V[0]）：
Closed polyline (N vertices, N segments, last = V[N-1]→V[0]):
  t ∈ [0, N)
  段索引 i = floor(t) % N
  Segment index i = floor(t) % N
  坐标 = lerp(V[i], V[(i+1)%N], s)
  Coordinate = lerp(V[i], V[(i+1)%N], s)
```

### 4.3 仿射变换组合 / Affine Transform Composition

```
// 先 scale → 再 rotate → 再 translate
// scale → rotate → translate
transform = T(tx, ty) · R(θ) · S(sx, sy)
           = [[sx·cosθ, -sy·sinθ, tx],
              [sx·sinθ,  sy·cosθ, ty]]

// 应用于点 / Applied to point
x' = a·x + b·y + tx
y' = c·x + d·y + ty
```

### 4.4 DXF 颜色匹配 / DXF Color Matching

```
ACI → Hex RGB（标准映射表子集 / Standard mapping subset）：
  1  → #FF0000  (红 / Red)
  2  → #FFFF00  (黄 / Yellow)
  3  → #00FF00  (绿 / Green)
  4  → #00FFFF  (青 / Cyan)
  5  → #0000FF  (蓝 / Blue)
  6  → #FF00FF  (紫 / Magenta)
  7  → #FFFFFF  (白/黑 / White/Black)
  8  → #808080  (灰 / Gray)
  9  → #C0C0C0  (亮灰 / Light Gray)
  10 → #FF0000  (红, 浅 / Red, light)
  ⋮

DXF 颜色编号 0 = ByBlock
DXF 颜色编号 256 = ByLayer
```

## 5. 质量与校验 / Quality & Validation

### 5.1 GSGI 文件自检规则 / GSGI File Self-Check Rules

发布前运行以下校验：
Run the following checks before release:

| 规则 / Rule | 说明 / Description |
|------|------|
| JSON Schema 校验 / Schema Validation | 结构合法、必填字段完整 / Structural validity, required fields complete |
| 引用完整性 / Reference Integrity | 所有 `*_ref` 字段指向存在的 id / All `*_ref` fields point to existing ids |
| 无循环引用 / No Circular References | groups、xref 无循环 / No circular refs in groups, xref |
| 块无嵌套 block_ref / No Nested block_ref in Blocks | 块内无 block_ref / xref / No block_ref/xref inside blocks |
| `properties.naming` 合规 / Naming Compliance | 所有 id 匹配 `id_pattern`，无 `reserved_prefix` 开头 / All ids match `id_pattern`, no `reserved_prefix` |
| param_pt 类型正确 / Correct param_pt Type | `curve_ref` 指向的实体是曲线类型 / Referenced entity is a curve type |
| 坐标系角度范围 / Coordinate Rotation Range | rotation 在合理范围 / Rotation in valid range |
